The Heimdall Helm chart supports multiple ingress controllers for exposing Heimdall to external traffic.

::tip
**Local Development?** See [Local Development Setup](/development/local-setup) for automated ingress setup with kind/minikube clusters.
::

## Quick Start

Enable ingress with minimal configuration:

```yaml
heimdall:
  ingress:
    enabled: true
    className: "traefik"  # or "nginx", "kong", "alb", etc.
    hosts:
      - host: heimdall.example.com
        paths:
          - path: /
            pathType: Prefix
```

With HTTPS using cert-manager:

```yaml
heimdall:
  ingress:
    enabled: true
    className: "traefik"
    annotations:
      cert-manager.io/cluster-issuer: "letsencrypt-prod"
    hosts:
      - host: heimdall.example.com
        paths:
          - path: /
    tls:
      - secretName: heimdall-tls
        hosts:
          - heimdall.example.com
```

## Supported Ingress Controllers

| Controller | Best For | Status | Setup Complexity |
|------------|----------|--------|------------------|
| **Traefik** | General use, modern features | ✅ Recommended | Low |
| **Nginx** | Mature deployments | ⚠️ Retiring March 2026 | Low |
| **AWS ALB** | Amazon EKS | ✅ Active | Medium |
| **GCP Load Balancer** | Google GKE | ✅ Active | Medium |
| **Azure App Gateway** | Azure AKS | ✅ Active | Medium |
| **Kong** | API gateway features | ✅ Active | Medium |

## When to Use Each Controller

### Traefik (Recommended)

**Use when:**
- ✅ Starting a new deployment
- ✅ Want modern features (middleware, Gateway API)
- ✅ Need dynamic configuration updates
- ✅ Built-in observability important

**Avoid when:**
- ❌ Already using another controller successfully
- ❌ Need specific Nginx features

### Nginx

**Use when:**
- ✅ Already deployed and working
- ✅ Team familiar with Nginx
- ✅ Legacy requirements

**Avoid when:**
- ❌ Starting new deployments (retiring March 2026)
- ❌ Need long-term support

### AWS ALB

**Use when:**
- ✅ Running on Amazon EKS
- ✅ Want native AWS integration
- ✅ Using AWS Certificate Manager (ACM)
- ✅ Need WAF integration

**Avoid when:**
- ❌ Not on AWS
- ❌ Multi-cloud deployment

### GCP Load Balancer

**Use when:**
- ✅ Running on Google GKE
- ✅ Want managed certificates
- ✅ Using Google Cloud Armor
- ✅ Need global load balancing

**Avoid when:**
- ❌ Not on GCP
- ❌ Multi-cloud deployment

### Azure App Gateway

**Use when:**
- ✅ Running on Azure AKS
- ✅ Want native Azure integration
- ✅ Need Azure WAF
- ✅ Using Azure Key Vault

**Avoid when:**
- ❌ Not on Azure
- ❌ Multi-cloud deployment

### Kong

**Use when:**
- ✅ Need API gateway features
- ✅ Want advanced plugins (rate limiting, auth)
- ✅ Have Kong expertise
- ✅ Enterprise features needed

**Avoid when:**
- ❌ Simple ingress needs
- ❌ No API gateway requirements

## Configuration Reference

Basic values.yaml structure:

```yaml
heimdall:
  ingress:
    # Enable ingress
    enabled: false

    # Ingress controller class
    className: "traefik"

    # Controller-specific annotations
    annotations: {}
      # cert-manager.io/cluster-issuer: "letsencrypt-prod"

    # Hostnames and paths
    hosts:
      - host: heimdall.local
        paths:
          - path: /
            pathType: Prefix

    # TLS configuration
    tls: []
      # - secretName: heimdall-tls
      #   hosts:
      #     - heimdall.example.com
```

## Common Scenarios

### Local Development

Disable ingress, use port-forward:

```yaml
heimdall:
  ingress:
    enabled: false
```

Access via:
```bash
kubectl port-forward -n heimdall svc/heimdall 8080:3000
```

### Production Internet-Facing

Traefik with automatic HTTPS:

```yaml
heimdall:
  ingress:
    enabled: true
    className: "traefik"
    annotations:
      cert-manager.io/cluster-issuer: "letsencrypt-prod"
      traefik.ingress.kubernetes.io/router.entrypoints: websecure
    hosts:
      - host: heimdall.example.com
        paths:
          - path: /
    tls:
      - secretName: heimdall-tls
        hosts:
          - heimdall.example.com
```

### Production Internal

Corporate CA certificate:

```yaml
heimdall:
  ingress:
    enabled: true
    className: "nginx"
    annotations:
      nginx.ingress.kubernetes.io/ssl-redirect: "true"
    hosts:
      - host: heimdall.internal.corp
        paths:
          - path: /
    tls:
      - secretName: corp-cert-secret
        hosts:
          - heimdall.internal.corp
```
