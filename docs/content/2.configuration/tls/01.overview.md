Heimdall supports HTTPS with two approaches:

1. **Automated** - cert-manager with Let's Encrypt (recommended for internet-facing)
2. **Manual** - Pre-created certificates (DoD PKI, corporate CA, purchased certs)

## Quick Start

**Automated (cert-manager)**:
```yaml
heimdall:
  ingress:
    enabled: true
    className: "traefik"
    annotations:
      cert-manager.io/cluster-issuer: "letsencrypt-prod"
    hosts:
      - host: heimdall.example.com
        paths:
          - path: /
    tls:
      - secretName: heimdall-tls  # Created automatically
        hosts:
          - heimdall.example.com
```

**Manual certificates**:
```bash
# First create secret with your certificate
kubectl create secret tls heimdall-tls \
  --cert=fullchain.crt \
  --key=tls.key \
  -n heimdall
```

```yaml
heimdall:
  ingress:
    enabled: true
    className: "nginx"
    hosts:
      - host: heimdall.mil.example
        paths:
          - path: /
    tls:
      - secretName: heimdall-tls  # Pre-created
        hosts:
          - heimdall.mil.example
```

## When to Use Automated (cert-manager)

**Best for:**
- ✅ Internet-facing deployments
- ✅ Public DNS accessible
- ✅ Port 80 accessible (for HTTP-01 challenge)
- ✅ Want automatic certificate renewal
- ✅ Using Let's Encrypt or other ACME provider

**Benefits:**
- Zero manual certificate management
- Automatic renewal (60 days before expiration)
- Free certificates from Let's Encrypt
- Supports wildcard certificates with DNS-01 challenge

**Not suitable for:**
- ❌ Air-gapped environments
- ❌ DoD/Government PKI requirements
- ❌ Corporate CA requirements

## When to Use Manual Certificates

**Best for:**
- ✅ DoD PKI certificates required
- ✅ Corporate CA certificates
- ✅ Internal/private deployments
- ✅ Air-gapped environments
- ✅ Purchased certificates (DigiCert, Comodo, etc.)

**Requirements:**
- Manual certificate renewal
- Certificate files (cert + private key)
- Full certificate chain (intermediates + root CA)

**Not suitable for:**
- ❌ Public internet deployments (use cert-manager instead)
- ❌ Frequent certificate rotation

## Prerequisites

### For Automated (cert-manager)

- Ingress controller installed (Traefik, Nginx, etc.)
- Ingress enabled in Heimdall chart
- DNS pointing to ingress controller
- Port 80 accessible for HTTP-01 challenge
- Or DNS provider credentials for DNS-01 wildcard certificates

### For Manual Certificates

- Certificate files (cert + key + chain)
- Certificate matches hostname (CN or SAN)
- Certificate not expired
- Private key unencrypted (or decrypt first with `openssl rsa`)
