## Design Principles

The Heimdall Helm chart follows industry-standard best practices established by [Helm](https://helm.sh/docs/chart_best_practices/), [Bitnami](https://github.com/bitnami/charts), and [Artifact Hub](https://artifacthub.io/).

### Secure by Default

- ClusterIP service (no external exposure without ingress)
- Ingress disabled by default
- Strict security contexts for containers
- Multiple secrets management approaches

### Production-Ready

- PodDisruptionBudget enabled by default
- Comprehensive health probes (startup, liveness, readiness)
- Zero-downtime rolling updates
- Resource limits and requests

### Flexible Configuration

- Three distinct secrets management approaches
- External database support
- Multiple authentication methods (local, LDAP, OIDC, OAuth)
- Extensive environment variable configuration

### Self-Contained

- Works out-of-box without prerequisites
- Embedded PostgreSQL via Bitnami subchart
- Automatic database migrations via Helm hooks
- Optional admin user creation

### Cloud-Native

- OCI registry support
- Bitnami subchart dependencies
- Values schema validation (JSON Schema Draft-07)
- Helm v4 compatible with Server-Side Apply (SSA)
