Get Heimdall up and running in 5 minutes with embedded PostgreSQL and local authentication.

## Prerequisites

- Kubernetes cluster (kind, minikube, or cloud cluster)
- Helm 3.12+ installed
- kubectl configured

## Quick Deploy

### 1. Add Helm Repository

```bash
helm repo add mitre https://mitre.github.io/heimdall-helm
helm repo update
```

### 2. Generate Secrets

```bash
# Download the secret generation script
curl -O https://raw.githubusercontent.com/mitre/heimdall-helm/main/generate-heimdall-secrets.sh
chmod +x generate-heimdall-secrets.sh

# Generate secrets file
./generate-heimdall-secrets.sh
```

This creates `heimdall-secrets.yaml` with random secrets.

### 3. Install Heimdall

```bash
helm install heimdall mitre/heimdall \
  --namespace heimdall \
  --create-namespace \
  --set-file heimdall.secretsFiles[0]=heimdall-secrets.yaml \
  --wait
```

### 4. Access Heimdall

```bash
# Port forward to access locally
kubectl port-forward -n heimdall svc/heimdall 3000:3000
```

Open http://localhost:3000 in your browser.

### 5. Login

Default credentials (created automatically):
- **Email**: `admin@heimdall.local`
- **Password**: Check the generated `heimdall-secrets.yaml` file for `ADMIN_PASSWORD`

## What You Get

This quickstart deploys:

- **Heimdall** StatefulSet (1 replica)
- **PostgreSQL** StatefulSet (Bitnami subchart)
- **Persistent volumes** for database storage
- **Local authentication** (email/password)
- **No ingress** (use port-forward or add later)
