The Heimdall Helm chart supports three approaches for managing sensitive credentials.

## Required Secrets

All approaches must provide these four secrets:

- **JWT_SECRET** - JWT signing secret (128 characters recommended)
- **API_KEY_SECRET** - API key encryption secret (66 characters recommended)
- **DATABASE_PASSWORD** - Database password (66 characters recommended)
- **ADMIN_PASSWORD** - Initial admin user password

## Three Approaches

| Approach | Best For | Complexity |
|----------|----------|------------|
| **Existing Secret** | Production deployments | Low |
| **File-based** | Local development | Low |
| **Inline** | CI/CD pipelines | Medium |

## When to Use Each

### Existing Secret (Production)

**Use when:**
- ✅ Using external secret management (ESO, Vault, Sealed Secrets)
- ✅ Cloud provider secret managers (AWS, GCP, Azure)
- ✅ Production or staging deployments
- ✅ Want automated secret rotation

**Configuration:**
```yaml
heimdall:
  existingSecret: "heimdall-production-secrets"
```

### File-based (Development)

**Use when:**
- ✅ Local development and testing
- ✅ Quick prototyping
- ✅ No external secret manager available

**Configuration:**
```yaml
heimdall:
  secretsFiles: ["env/heimdall-secrets.yaml"]
```

### Inline (CI/CD)

**Use when:**
- ✅ Automated deployments
- ✅ CI/CD pipelines (GitHub Actions, GitLab CI)
- ✅ Secrets injected from environment variables

**Configuration:**
```yaml
heimdall:
  secrets:
    JWT_SECRET: "from-ci-environment"
    API_KEY_SECRET: "from-ci-environment"
    DATABASE_PASSWORD: "from-ci-environment"
    ADMIN_PASSWORD: "from-ci-environment"
```

## Priority Order

If multiple approaches are configured, this is the precedence:

```
existingSecret > secrets > secretsFiles
```

Example:
```yaml
heimdall:
  existingSecret: "my-secret"  # This wins
  secretsFiles: ["env/secrets.yaml"]  # Ignored
  secrets:  # Ignored
    JWT_SECRET: "ignored"
```

## Generating Secrets

Use cryptographically secure random generators:

**JWT Secret (128 chars):**
```bash
openssl rand -hex 64
```

**API Key Secret (66 chars):**
```bash
openssl rand -hex 33
```

**Database Password (66 chars):**
```bash
openssl rand -hex 33
```

**All at once:**
```bash
echo "JWT_SECRET: $(openssl rand -hex 64)"
echo "API_KEY_SECRET: $(openssl rand -hex 33)"
echo "DATABASE_PASSWORD: $(openssl rand -hex 33)"
echo "ADMIN_PASSWORD: Admin_$(openssl rand -hex 8)"
```
